HERRAMIENTAS

  • WIT y GITHUB

    ¿Existe algún lugar o método que mantenga a salvo nuestros ficheros importantes a la hora de programar?
    Siendo que el alojamiento en espacios físicos como discos externos, el almacenamiento USB, discos duros extraíbles, etc; no es práctico y puesto que el alojamiento en sitios especializados como Dropbox o la nube no es que mantenga una confiabilidad del 100% y que es inviable para el desarrollo continuo de un archivo de código que deba ser actualizado o modificado continuamente… entonces… existe una ayuda al momento de almacenar, compartir y modificar nuestro código?
    No solo existe, sino que es una gran ayuda: los repositorios GIT; no es otro idioma, en realidad esta clase de repositorios son una copia local del código generado con un plus bastante importante y es que se pueden hacer muchas versiones para cada modificación, que en caso de error y nuestra aplicación no funciona es un salvavidas a tener en cuenta y no solo en ese caso; también si se quiere probar un cambio en el que se apliquen nuevas funcionalidades lo podemos hacer sin modificar ni arriesgar nuestro código funcional.
    Pero bueno GIT se mantiene en local y no soluciona el problema del caso de perdida o daño del dispositivo de almacenamiento…
    Bueno, pues permítanme presentarlos con la solución (tras un simple enlace con GIT) y se llama GITHUB con el cual se sincroniza a los repositorios GIT y los guarda en un ambiente en red.

    Con todo lo anterior ya se marca una diferencia, pero lo que realmente hace que a GIT tan eficiente son llamados commit y branch.
    El commit no es mas que una fotografía del proyecto con fecha y hora que describe los cambios hechos desde el ultimo commit, básicamente, se podría decir que es como una referencia para que quien vaya a modificar tu proyecto sepa en que ha cambiado desde la ultima vez que lo vio.
    Sin embargo cuando varias personas están trabajando en el mismo proyecto mediante la sincronización con GITHUB puede que sea difícil ponerse de acuerdo o el uso de los commit al principio, para solucionar esto GITHUB tiene los Branch que nos son mas que una herramienta que permite a los miembros del equipo trabajar en paralelo sobre el mismo proyecto, y experimentar que si funcionan pueden integrarse al código base( rama principal o master)

  • HTML

    El lenguaje de programación htlm (HyperTextMarkupLanguage) podría definirse mejor como un lenguaje de etiquetas con el que se puede construir documentos web o, propiamente dicho, paginas web de forma que los diferentes navegadores puedan entender nuestro contenido y lo puedan mostrar a los usuarios… y aunque no es el único que se puede usar, la inmensa mayoría de páginas están hechas en él.

    Ya aclarado que HTLM es un lenguaje de etiquetas o de “marcas” se debe decir que su última versión la llamada HTLM5 tiene algunas ventajas que le hacen una de las mejores herramientas a la hora de diseñar páginas web ya que a diferencia de sus versiones anteriores tiene una gran flexibilidad para estructurar y presentar, gráficamente hablando, páginas web. Desde la creación de animaciones hasta la construcción de aplicaciones de ambiente móvil; es HTLM5, en toda forma, una muestra de mejoría de cara a versiones anteriores sin dejar de lado las nociones básicas con lo cual un diseñador que haya aprendido o tenga nociones básicas de sus versiones anteriores encontrara en ello un pie de apoyo.

  • CSS3

    Creo que la definición mas acertada para CSS es el de una hoja de de estilo con mucho estilo, aunque su historia se debe al boom del internet y la creciente popularidad del lenguaje de etiquetas htlm y la necesidad de un estándar para la definición de los estilos. Por lo anterior y en medio de la guerra de los navegadores el organismo llamado w3c(world wide web consortium) que es el encargado de crear todos los estándares relacionados con la web, propuso la creación de un lenguaje de hojas de estilos específicos para el lenguaje HTLM, despues de mucho jaleo que no viene a cuento de las 9 propuestas que se presentaron solo se tuvieron en cuenta 2: la CHSS(Cascading HTLM Style Sheets) y la SSP(Stream-based Style Sheet Proposal). Los dos realizadores de estas dos propuestas se unieron y definieron un nuevo lenguaje que tomaba lo mejor de cada una y lo llamaron CSS (Cascading Style Sheets).

    El poder de CSS reside en su capacidad para definir la estética de un sitio web en un documento externo y es esto muy importante, ya que, esto mismo permite que modificando ese documento (hoja CSS) podamos cambiar la estética entera del sitio web, en pocas palabras, todo el sitio web puede variar totalmente de estética cambiando solo la CSS, sin tocar los documentos HTLM que la componen.
